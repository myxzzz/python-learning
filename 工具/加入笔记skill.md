## 📝 实际填充示例

### 例1：GroupBy 百分比化

**章节名称**：第 15 章：GroupBy 百分比化 - 自定义格式转换与动态列生成

**核心功能**：把 groupby 后的小数（0.1234）一键转成百分比字符串（12.34%）

**适用场景**：

- 需要给老板看报表时（老板看不懂小数）
- 要导出 Excel 时（格式要规范）
- 需要快速对比多个类别的占比

**难度等级**：⭐⭐

**核心理念**：
写一个通用的"转换函数"，然后用 `apply` 把它作用到表的每一行。这样不用逐行手工改，也不用复杂的 `for` 循环。

**业务场景示例**：

- 北京奶茶店的"位置类型"占比（商务区 vs 景区 vs 其他）
- 消费者性别分布
- 产品类别的销售占比

**核心代码片段**：

```python
def format_percentage(value, total):
    """转换函数：防止分母为0"""
    if total == 0: 
        return "0.00%"
    return f"{(value / total * 100):.2f}%"

# 应用到表的每一行
for col in df_table1.columns[:-1]:  # 排除合计列
    df_table1[f'{col}_占比'] = df_table1.apply(
        lambda row: format_percentage(row[col], row['合计']), 
        axis=1
    )
```

**容易踩的坑**：

- ❌ 忘记加 `if total == 0` 检查，遇到空城区会报 ZeroDivisionError
- ❌ `axis=1` 写反了会变成按列应用，导致行列错乱
- ❌ 新列名和原列名重复，会覆盖原数据

**扩展玩法**：

- 如果不要百分比符号，改成 `f"{(value / total * 100):.2f}"`
- 如果想要四舍五入到整数，改成 `f"{round(value / total * 100)}%"`

## 💡 笔记添加的"黄金法则"

1. **真实使用优先**：只记录你真的在项目里用过的技巧，不要凭空想象
2. **一个技巧一章**：不要把多个无关的技巧混在一起
3. **从痛点出发**：先描述"我遇到的问题"，再说"这个技巧怎么解决"
4. **代码要能跑**：粘贴的代码片段必须能在你的环境里直接运行
5. **记录坑点**：这是最值钱的部分，初学者最容易踩的坑要特别标注
