# 🚨 99_常见错误速查\n\n> 遇到报错？按错误症状找，这里有解决方案。\n\n---\n\n## 错误 1：KeyError\n\n**症状**：\n```\nKeyError: 'column_name'\n```\n\n**原因**：\n- 你访问的列名不存在（可能写错了列名，或者大小写不对）\n- 做了某种操作（比如 `groupby`）后，原列名变成了索引\n\n**解决方案**：\n\n```python\n# 1️⃣ 检查列名是否存在\nprint(df.columns)  # 看看实际的列名\n\n# 2️⃣ 注意大小写、空格\n# ❌ 错误：df['销售额']（如果实际列名是\" 销售额 \"有空格）\n# ✅ 修正：\ndf.columns = df.columns.str.strip()  # 先清理列名\ndf['销售额']\n\n# 3️⃣ 如果是 groupby 后的索引问题\n# ❌ 错误：\nresult = df.groupby('品牌')['销量'].sum()\nresult['品牌']  # 报错！品牌已经是索引了\n\n# ✅ 修正：\nresult = df.groupby('品牌')['销量'].sum().reset_index()\nresult['品牌']  # ✓\n```\n\n---\n\n## 错误 2：TypeError 或 ValueError\n\n### 症状 A：\"The truth value of a Series is ambiguous\"\n\n**原因**：`loc` 多条件用了 `and`/`or` 而不是 `&`/`|`\n\n**解决方案**：\n\n```python\n# ❌ 错误\ndf.loc[df['价格'] > 100 and df['品牌'] == 'Apple']\n\n# ✅ 修正\ndf.loc[(df['价格'] > 100) & (df['品牌'] == 'Apple')]\n```\n\n---\n\n### 症状 B：\"could not convert string to float\"\n\n**原因**：想把文本转成数字，但里面混有\"无效\"的文本（如\"暂无\"、\"N/A\"）\n\n**解决方案**：\n\n```python\n# ❌ 错误（会报错）\ndf['价格'].astype('float64')\n\n# ✅ 修正（用 pd.to_numeric，脏数据变 NaN）\ndf['价格'] = pd.to_numeric(df['价格'], errors='coerce')\n```\n\n---\n\n## 错误 3：merge 后行数爆炸\n\n**症状**：\n```\n原本 df_A 有 100 行，df_B 有 50 行\nmerge 后变成 5000 行！\n```\n\n**原因**：\nJOIN 的字段在两表中有重复值，导致笛卡尔积\n\n**解决方案**：\n\n```python\n# 第 1 步：检查重复情况\nprint(df_A['品牌'].value_counts())\nprint(df_B['品牌'].value_counts())\n\n# 第 2 步：如果某个品牌在 B 表有多行，先去重\ndf_B_clean = df_B.drop_duplicates(subset=['品牌'], keep='first')\n\n# 第 3 步：再 merge\nresult = pd.merge(df_A, df_B_clean, on='品牌', how='left')\n```\n\n---\n\n## 错误 4：groupby 后的索引问题\n\n**症状 A**：groupby 结果无法和其他表 merge\n\n```python\n# ❌ 错误\nresult = df.groupby('品牌')['销量'].sum()\npd.merge(result, other_df, on='品牌')  # 报错！\n\n# ✅ 修正\nresult = df.groupby('品牌')['销量'].sum().reset_index()\npd.merge(result, other_df, on='品牌')  # ✓\n```\n\n---\n\n**症状 B**：groupby 后想用 result['品牌'] 访问列\n\n```python\n# ❌ 错误\nresult = df.groupby('品牌')['销量'].sum()\nprint(result['品牌'])  # KeyError，因为'品牌'是索引\n\n# ✅ 修正\nresult = df.groupby('品牌')['销量'].sum().reset_index()\nprint(result['品牌'])  # ✓\n```\n\n---\n\n## 错误 5：to_datetime 转换失败\n\n**症状**：\n```\nParserError: unknown datetime format\n或\nResult contains NaT (Not a Time)\n```\n\n**原因**：\n日期格式不统一或格式与指定的不匹配\n\n**解决方案**：\n\n```python\n# 方案 1：指定格式\ndf['日期'] = pd.to_datetime(df['日期'], format='%Y-%m-%d')\n\n# 方案 2：让 Pandas 自动推断格式\ndf['日期'] = pd.to_datetime(df['日期'], infer_datetime_format=True)\n\n# 方案 3：用 errors='coerce'，无法转的变成 NaT\ndf['日期'] = pd.to_datetime(df['日期'], errors='coerce')\n```\n\n---\n\n## 错误 6：.str 方法无法使用\n\n**症状**：\n```\nAttributeError: 'float' object has no attribute 'str'\n```\n\n**原因**：\n列里混有非字符串类型（如 NaN、数字），`.str` 方法无法处理\n\n**解决方案**：\n\n```python\n# ❌ 错误\ndf['名字'].str.upper()  # 如果某个值是 NaN，报错\n\n# ✅ 修正 1：先填充 NaN\ndf['名字'] = df['名字'].fillna('')\ndf['名字'].str.upper()\n\n# ✅ 修正 2：转成字符串\ndf['名字'] = df['名字'].astype('str')\ndf['名字'].str.upper()\n\n# ✅ 修正 3：用 na=False 避免处理 NaN\ndf['名字'].str.contains('某词', na=False)\n```\n\n---\n\n## 错误 7：文件读取乱码\n\n**症状**：\n```\nCSV 打开看到中文变成 ???\n或\nExcel 里的中文在 Python 里乱码\n```\n\n**原因**：\n编码格式不对（UTF-8、GBK、GB2312 等）\n\n**解决方案**：\n\n```python\n# 方案 1：读 CSV 时指定编码\ndf = pd.read_csv('file.csv', encoding='utf-8')\n# 如果报错，试试\ndf = pd.read_csv('file.csv', encoding='gbk')\ndf = pd.read_csv('file.csv', encoding='latin1')\n\n# 方案 2：保存 CSV 时用 utf-8-sig（Excel 友好）\ndf.to_csv('output.csv', encoding='utf-8-sig', index=False)\n\n# 方案 3：读 Excel\ndf = pd.read_excel('file.xlsx')  # Excel 通常自动处理编码\n```\n\n---\n\n## 错误 8：pivot_table 和 melt 的列标签问题\n\n**症状**：\n```\npivot 后再 melt，多出一个奇怪的列标签\n或\nmelt 后某列的名字跟预期不同\n```\n\n**原因**：\npivot 后有隐形的列名标签没有清理\n\n**解决方案**：\n\n```python\n# ❌ 错误\ndf_wide = df.pivot_table(...)\ndf_long = df_wide.melt(...)  # 可能多出一列\n\n# ✅ 修正\ndf_wide = df.pivot_table(...)\ndf_wide = df_wide.reset_index()                  # 把行标签变回列\ndf_wide = df_wide.rename_axis(None, axis=1)     # 清理列标签\ndf_long = df_wide.melt(...)  # ✓\n```\n\n---\n\n## 错误 9：数据库连接错误\n\n**症状**：\n```\noperational error: could not connect to server\n或\npsycopg2.ProgrammingError: column \"xxx\" does not exist\n```\n\n**原因**：\n- 连接信息错误（主机、端口、用户名、密码）\n- SQL 写法有问题（列名大小写、引号）\n\n**解决方案**：\n\n```python\n# 1️⃣ 检查连接信息\nconn = psycopg2.connect(\n    host='localhost',      # 检查主机\n    port='5432',           # 检查端口\n    database='my_db',      # 检查数据库名\n    user='postgres',       # 检查用户名\n    password='xxxxx'       # 检查密码\n)\n\n# 2️⃣ SQL 中列名用双引号（PostgreSQL 区分大小写）\ndf = pd.read_sql('SELECT \"用户ID\", \"销售额\" FROM table', conn)\n\n# 3️⃣ 确保 commit() 了（DDL 操作后必须提交）\ncur.execute(\"CREATE TABLE xxx AS SELECT ...\")\nconn.commit()  # ← 不能忘\n```\n\n---\n\n## 错误 10：os.listdir() 包括文件夹\n\n**症状**：\n```\n你想读 .csv 文件，结果报错说\"某文件夹\"不是文件\n```\n\n**原因**：\nos.listdir() 返回文件和文件夹的混合列表\n\n**解决方案**：\n\n```python\n# ❌ 错误\nfiles = os.listdir(path)\nfor f in files:\n    df = pd.read_csv(os.path.join(path, f))  # 如果 f 是文件夹，报错\n\n# ✅ 修正 1：按后缀过滤\nfiles = [f for f in os.listdir(path) if f.endswith('.csv')]\n\n# ✅ 修正 2：检查是否为文件\nfiles = [f for f in os.listdir(path) if os.path.isfile(os.path.join(path, f))]\n```\n\n---\n\n## 快速查找 Cheat Sheet\n\n| 症状 | 查看这个错误号 |\n|------|----------------|\n| 列名找不到 | 错误 1：KeyError |\n| 多条件用 and/or 了 | 错误 2：TypeError 症状 A |\n| 数字转换失败 | 错误 2：TypeError 症状 B |\n| merge 后行数爆炸 | 错误 3 |\n| groupby 后无法访问列 | 错误 4 |\n| 日期转换失败 | 错误 5 |\n| .str 方法报错 | 错误 6 |\n| 中文乱码 | 错误 7 |\n| pivot/melt 列名乱 | 错误 8 |\n| 数据库连接错误 | 错误 9 |\n| 读文件夹下的文件出错 | 错误 10 |\n\n---\n\n*最后更新：2026-04-19*\n*遇到新的错误？把解决方案加到这里～*\n
