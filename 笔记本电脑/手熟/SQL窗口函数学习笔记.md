# SQL 窗口函数 (Window Functions) 核心笔记

> 💡 **核心逻辑**：窗口函数是在每一行数据旁边开一个“上帝视角”的小窗户。它**不会减少行数**（不改变表结构），但能让你在这一行直接看到“全表”或“组内”的统计值。

---

## 1. 核心公式 (万能模板)

```sql
函数() OVER(PARTITION BY 分组列 ORDER BY 排序列 顺序)
```

- **函数()**：你要算什么？(`SUM`, `AVG`, `COUNT`, `RANK`, `ROW_NUMBER`)
- **OVER()**：**标志着这是一个窗口函数！** 没有它就是普通聚合。
- **PARTITION BY**：**“圈地”**。对应 Pandas 的 `groupby`。不写它看全表，写了它看组内。
- **ORDER BY**：**“排队”**。对应 Pandas 的 `sort_values`。**只要写了它，计算就会变成“流动的（Rolling）”**。

---

## 2. 三大常用场景

### 场景一：占比计算 (分子/分母)

**需求**：计算每个型号在全市场的产值占比。

```sql
SELECT 
    机型, 
    产值,
    -- SUM(产值) OVER() 是全局总和，每行都一样
    ROUND(产值 * 100.0 / SUM(产值) OVER(), 2) AS 市场占比
FROM df;
```

### 场景二：组内对比 (单行 vs 组均值)

**需求**：查看这款电脑比所属品牌的平均价高出多少。

```sql
SELECT 
    机型, 品牌, 价格,
    -- AVG(价格) OVER(PARTITION BY 品牌) 让每行都能看到"娘家"的平均价
    价格 - AVG(价格) OVER(PARTITION BY 品牌) AS 价格偏离度
FROM df;
```

⚠️ **注意**：这里一定不能写 `ORDER BY`，否则平均值会变成“滚动平均”。

### 场景三：存钱罐/滚雪球 (累计求和)

**需求**：计算前几名品牌累计占了多少市场份额。

```sql
SELECT 
    品牌, 市场占比,
    -- 重点：有了 ORDER BY，SUM 就会从第一行一直加到当前行
    SUM(市场占比) OVER(ORDER BY 市场占比 DESC) AS 累计份额
FROM brand_stats;
```

💡 **对照**：对应 Pandas 的 `.cumsum()`。

## 3. 排名函数全家桶

| 函数 | 效果 | 例子 (价格 5000, 5000, 4000) |
| :--- | :--- | :--- |
| **ROW_NUMBER()** | 严格的一行一个号 | 1, 2, 3 |
| **RANK()** | 并列跳号 | 1, 1, 3 |
| **DENSE_RANK()** | 并列不跳号 (常用) | 1, 1, 2 |

---

## 4. 终极避坑指南

1. **中英文符号**：SQL 必须用英文半角逗号 `,`，绝对不能用全角 `，`。
2. **除法陷阱**：计算比例时分子记得 `* 100.0`，否则整数相除结果是 0。
3. **聚合嵌套**：计算“品牌占比”时，分母是 `SUM(SUM(A)) OVER()`，内层 SUM 算品牌，外层 SUM 算全市场。
